   - 这与 `delivery.mode=announce` 冲突，且隔离会话调 message 不可靠

### 教训更新
- cron prompt 不应包含手动调 message 的指令 → 系统层 announce 投递是唯一可靠路径
- 隔离会话调 message 工具虽然能拿到 messageId，但消息可能被微信折叠/吞掉
- delivery.mode 会在某些条件下被重置（可能是 gateway 重启、jobs.json 重建），需要定期检查
