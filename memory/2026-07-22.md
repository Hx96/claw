
## 07:20 AI热点推送未送达排查

**现象：**
- 07:30 AI热点推送 cron 正常执行（耗时62.9s），status=ok, delivered=true
- 用户微信未收到消息
- 07:34 手动触发 Builders 简报（1054字符/1321字节），用户正常收到

**根因分析：**
- AI热点内容 1853字符 / 2855字节 → 未收到
- Builders简报 1054字符 / 1321字节 → 收到
- **判断：微信企业消息存在长度阈值，~2000字节以上可能被折叠/丢弃**
- `delivered=true` 表示 OpenClaw → 微信 API 调用成功，但微信侧可能静默丢弃长消息
- 这与 06-27/06-30 的结论一致：长消息在微信侧不可靠

**07:20 推送当前流程：**
1. cron 07:30 触发（schedule: 30 7 * * *，注意schedule已从20改到30）
2. isolated session 中 AI 执行 8 个 opencli 命令收集数据
3. AI 筛选 5-8 条 AI 新闻，格式化输出（markdown）
4. delivery.mode=announce，系统层投递完整文本到微信
5. 微信 API 返回成功（delivered=true），但长消息可能被折叠

**改进方向：**
- 热点推送内容需要压缩到 ~1300字符以内（与Builders一致）
- 或者像 Builders 一样分段发送
- prompt 中加字数硬限制

**教训更新：**
- MEMORY.md 教训#14 需要细化：不只是 Builders 长消息有问题，AI热点同样问题
- 阈值大约在 2000 字节（UTF-8），约 1500 中英混合字符
- cron announce 投递 + delivered=true 仍不可靠，根因在微信侧
