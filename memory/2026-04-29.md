# 2026-04-29

## Memory问题修复（08:03）

**问题：** memory日志严重断档，04-27修过一次但04-28、04-29又断了。

**根因分析：**
1. `openclaw.json` 中从未配置 `heartbeat` 字段，heartbeat机制从未启动
2. HEARTBEAT.md写了检查规则但等于空文——没人触发它
3. AGENTS.md要求"对话结束时写memory"，但session过期/compaction/新会话覆盖时memory就丢了
4. 单靠"对话结束时"写memory是不可靠的——对话并不总是有明确的"结束"时刻

**修复：**
- 在 `agents.defaults.heartbeat` 添加完整配置：
  - every: "2h" — 每2小时一次
  - target: "none" — 不主动发消息，内部检查
  - lightContext: true — 只注入HEARTBEAT.md，省token
  - isolatedSession: true — 不带历史上下文
  - activeHours: 07:00-23:00 Asia/Shanghai
- 更新HEARTBEAT.md：memory日志检查 + MEMORY.md周更新
- Gateway热加载确认：`config hot reload applied (agents.defaults.heartbeat)`
- 手动触发 `openclaw system event --text "memory check" --mode now` 验证event机制正常

**教训第11条：** heartbeat配置不在openclaw.json里就等于没配置。HEARTBEAT.md写了规则但没有heartbeat机制触发，就是空文。配置和运行机制必须同时存在。
